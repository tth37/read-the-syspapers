OSDI '25 共收录 **53 篇论文**，保持单一 track 的编排。整届会议的重心明显向机器学习系统倾斜——53 篇里有 16 篇直接讨论大模型服务、训练或 ML kernel 工具链——但 OSDI 的经典版图仍然完整，内存/存储、分布式事务、形式化验证与内核工作都保持了充足的比例。今年两股相互拉扯的力量尤为突出：一边是**精细化分解**（解聚合内存、CXL pod、分层内存、纠删码放置、被拆分角色的 NIC 收发环），另一边是**投机式重组**（先按猜测前进、必要时再回滚的协议与检索）。程序里罕见地出现了两篇量子 OS 论文和一篇关于串行性能优化的方法论论文，让这届 OSDI 对"系统工作怎么做"而非只关心"做了什么"的自我反思尤其醒目。

## 主题

- **大模型服务与训练已经成为一等的系统工作负载。** 服务侧的代表工作包括 [BlitzScale](../papers/osdi-2025/blitzscale-fast-and-live-large-model-autoscaling-with-o-1-host-caching.md) 用权重多播换来 O(1) 规模的自动扩缩容、[NanoFlow](../papers/osdi-2025/nanoflow-towards-optimal-large-language-model-serving-throughput.md) 通过自动搜索的 nano-batch 流水把计算/内存/网络 kernel 重叠在单张 GPU 内、[WaferLLM](../papers/osdi-2025/waferllm-large-language-model-inference-at-wafer-scale.md) 把 prefill 与 decode 映射到晶圆级 mesh 加速器、[DecDEC](../papers/osdi-2025/decdec-a-systems-approach-to-advancing-low-bit-llm-quantization.md) 用 CPU 侧残差通道让低比特量化的质量损失几乎被隐藏。训练侧同样密集：[WLB-LLM](../papers/osdi-2025/wlb-llm-workload-balanced-4d-parallelism-for-large-language-model-training.md) 重新平衡变长序列下的 4D 并行、[ZEN](../papers/osdi-2025/zen-empowering-distributed-training-with-sparsity-driven-data-synchronization.md) 为稀疏梯度同步搜索接近最优的计划、[Understanding Stragglers](../papers/osdi-2025/understanding-stragglers-in-large-model-training-using-what-if-analysis.md) 在训练 trace 上建立 what-if 仿真量化落后节点代价、[TrainCheck](../papers/osdi-2025/training-with-confidence-catching-silent-errors-in-deep-learning-training-with-automated.md) 则在指标漂移之前捕捉静默错误。

- **加速器 kernel 被当成编译器 IR 对待，而不是黑盒二进制。** [KPerfIR](../papers/osdi-2025/kperfir-towards-a-open-and-compiler-centric-ecosystem-for-gpu-kernel-performance-tooling.md) 把 GPU 剖析表达成 Triton IR pass、[NEUTRINO](../papers/osdi-2025/neutrino-fine-grained-gpu-kernel-profiling-via-programmable-probing.md) 在汇编层注入可编程探针、[Mirage](../papers/osdi-2025/mirage-a-multi-level-superoptimizer-for-tensor-programs.md) 在 kernel/block/thread 三个层次做搜索并用概率式验证确认语义、[QiMeng-Xpiler](../papers/osdi-2025/qimeng-xpiler-transcompiling-tensor-programs-for-deep-learning-systems-with-a-neural.md) 用 LLM 生成骨架加 SMT 修复跨 CUDA/HIP/BANG/VNNI 转译，而 [PipeThreader](../papers/osdi-2025/pipethreader-software-defined-pipelining-for-efficient-dnn-execution.md) 则在显式 sTask 图上软件调度 TMA/TensorCore/CUDA-core 三条流水。

- **内存与存储的方向从"更多"转向"更细的控制"。** [FineMem](../papers/osdi-2025/finemem-breaking-the-allocation-overhead-vs-memory-waste-dilemma-in-fine-grained.md) 把 RDMA 的注册、保护与分配三件事拆开，[Tigon](../papers/osdi-2025/tigon-a-distributed-database-for-a-cxl-pod.md) 只把跨主机活跃元组放进 CXL 共享内存并用 DB 感知的软件一致性免去 2PC，[AOL](../papers/osdi-2025/tiered-memory-management-beyond-hotness.md) 用考虑延迟与 MLP 的 impact 代替单纯的 hotness，[EMT](../papers/osdi-2025/emt-an-os-framework-for-new-memory-translation-architectures.md) 把 Linux 的地址翻译重构成可插拔的 MMU driver，[Okapi](../papers/osdi-2025/okapi-decoupling-data-striping-and-redundancy-grouping-in-cluster-file-systems.md) 把条带宽度与纠删码分组宽度解耦，[Nos](../papers/osdi-2025/stripeless-data-placement-for-erasure-coded-in-memory-storage.md) 直接靠组合设计选副本位置、去掉条带的概念。

- **形式化验证走出实验室，进入运行时检查、溯源证明与不可信代码的确定性。** [Basilisk](../papers/osdi-2025/basilisk-using-provenance-invariants-to-automate-proofs-of-undecidable-protocols.md) 借助 atomic sharding 把不可判定分布式协议的归纳不变量合成做成了 Dafny 上可用的流程，[PoWER](../papers/osdi-2025/power-never-corrupts-tool-agnostic-verification-of-crash-consistency-and-corruption.md) 同时在 Verus 和 Dafny 上验证 PM 崩溃一致性与数据损坏处理，[T2C](../papers/osdi-2025/deriving-semantic-checkers-from-tests-to-detect-silent-failures-in-production-distributed.md) 把已有系统测试改造成运行时语义检查器，[DeCl](../papers/osdi-2025/deterministic-client-enforcing-determinism-on-untrusted-machine-code.md) 在不引入可信 JIT 的前提下静态验证 x86-64/Arm64 机器码的确定性执行，[Omniglot](../papers/osdi-2025/building-bridges-safe-interactions-with-foreign-languages-through-omniglot.md) 则让 Rust 调用外部库既保持可靠性又保留零拷贝共享。

- **异构调度和剖析开始跟上硬件多样性。** [XSched](../papers/osdi-2025/xsched-preemptive-scheduling-for-diverse-xpus.md) 用统一的 XQueue 抽象在 GPU、NPU、ASIC、FPGA 之间做抢占式调度，[Tintin](../papers/osdi-2025/tintin-a-unified-hardware-performance-profiling-infrastructure-to-uncover-and-manage.md) 把硬件计数器的多路复用误差当成一等的运行时不确定性处理，而 [QOS](../papers/osdi-2025/qos-quantum-operating-system.md) 与 [HyperQ](../papers/osdi-2025/quantum-virtual-machines.md) 把"可调度的云资源"这套思路延伸到 QPU 上，标志着一个正在生长的量子系统子方向。

## 值得关注的趋势

- **先投机，再回滚。** 跨多个层次反复出现的同一手法：[Mako](../papers/osdi-2025/mako-speculative-distributed-transactions-with-geo-replication.md) 在 WAN 复制完成前就对跨分片事务做投机认证、[SpecLog](../papers/osdi-2025/low-end-to-end-latency-atop-a-speculative-shared-log-with-fix-ante-ordering.md) 在全局排序完成前就交付共享日志记录、[PipeANN](../papers/osdi-2025/achieving-low-latency-graph-based-vector-search-via-aligning-best-first-search-algorithm.md) 在图遍历中流水线式投机地发起 SSD 读、[Tigon](../papers/osdi-2025/tigon-a-distributed-database-for-a-cxl-pod.md) 则用 DB 感知的软件一致性免去 2PC。四者都在用少见的回滚代价换取常态下的更低延迟。

- **角色分解优先于引入新机制。** 几篇论文的共同动作是把历史上被一个接口绑在一起的两个职责拆开：[FineMem](../papers/osdi-2025/finemem-breaking-the-allocation-overhead-vs-memory-waste-dilemma-in-fine-grained.md) 拆开 RDMA 注册、保护与分配，[rxBisect](../papers/osdi-2025/disentangling-the-dual-role-of-nic-receive-rings.md) 把 NIC 的 Rx ring 拆成"分配环"和"接收环"，[Okapi](../papers/osdi-2025/okapi-decoupling-data-striping-and-redundancy-grouping-in-cluster-file-systems.md) 把条带宽度和冗余组宽度解耦，[Omniglot](../papers/osdi-2025/building-bridges-safe-interactions-with-foreign-languages-through-omniglot.md) 把 FFI 的安全性拆成 sandbox、typestate 与 scope 三个互补维度。

- **从已有产物中挖出检查机制。** 与其手写不变量，不如让系统自己生成：[T2C](../papers/osdi-2025/deriving-semantic-checkers-from-tests-to-detect-silent-failures-in-production-distributed.md) 从已有测试里切出断言、[TrainCheck](../papers/osdi-2025/training-with-confidence-catching-silent-errors-in-deep-learning-training-with-automated.md) 从示例训练流程中学出上下文相关的不变量、[Basilisk](../papers/osdi-2025/basilisk-using-provenance-invariants-to-automate-proofs-of-undecidable-protocols.md) 从 provenance 中合成归纳不变量、[Paralegal](../papers/osdi-2025/paralegal-practical-static-analysis-for-privacy-bugs.md) 在所有权敏感类型近似得到的 PDG 上对标记化的隐私策略做检查。

- **网络与 DMA 路径在做"减法"。** [Söze](../papers/osdi-2025/s-ze-one-network-telemetry-is-all-you-need-for-per-flow-weighted-bandwidth-allocation-at.md) 仅用一个 per-packet 最大队列延迟信号就实现了去中心化的每流加权 max-min 带宽分配，[rxBisect](../papers/osdi-2025/disentangling-the-dual-role-of-nic-receive-rings.md) 通过隔离突发吸收环缩小 DDIO 工作集，[VIO](../papers/osdi-2025/to-pri-or-not-to-pri-that-s-the-question.md) 在 VirtIO 请求上做 snoop，让 I/O 缺页脱离 DMA 关键路径。三篇的共同动作是"去机制"而不是"加机制"。

## 必读推荐

- **[Basilisk](../papers/osdi-2025/basilisk-using-provenance-invariants-to-automate-proofs-of-undecidable-protocols.md)** —— 本届 Best Paper；atomic sharding 把不可判定分布式协议的全局归纳不变量合成变成了 Dafny 上可落地的工作流。
- **[FineMem](../papers/osdi-2025/finemem-breaking-the-allocation-overhead-vs-memory-waste-dilemma-in-fine-grained.md)** —— 解聚合内存的 allocator 底座：每次分配平均约 1.3 次 CAS，对比此前 one-sided 设计的 45 次；在相对静态预分配仅 2.5–4.1% 开销下带来 2.25–2.8× 的利用率提升，可以作为 malloc、KV、swap 系统共用的底层构件。
- **[NanoFlow](../papers/osdi-2025/nanoflow-towards-optimal-large-language-model-serving-throughput.md)** —— 自动搜索出的 nano-batch 让计算/内存/网络 kernel 在单张 GPU 内真正互相重叠，是 LLM 服务栈可以直接借鉴的吞吐优化。
- **[Mako](../papers/osdi-2025/mako-speculative-distributed-transactions-with-geo-replication.md)** —— "投机加回滚"思路的一次清晰概括：跨分片事务在 WAN 复制完成前就提交，由向量时钟与 vector watermark 在分片故障后界定回滚范围。
- **[KPerfIR](../papers/osdi-2025/kperfir-towards-a-open-and-compiler-centric-ecosystem-for-gpu-kernel-performance-tooling.md)** —— 社区基础设施：把 GPU kernel 剖析改造成 Triton IR pass，使得 FA3 级别的优化不再被厂商黑箱工具所遮蔽。

## 数据概览

- **已综述论文数：** 53（覆盖整个单 track 程序）。
- **分类分布：** 大模型系统 8，机器学习编译与 GPU kernel 8，数据库与向量检索 8，内存与存储 8，网络与虚拟化 7，内核/OS/隔离 6，形式化验证与安全 6，量子计算 2。
- **ML vs. 经典系统：** llm-systems + ml-compilers-and-gpu-kernels 合计 16 篇（约 30%）；另有约 4 篇在其他 track 里带 ML 属性（FuseLink、Compass、PipeANN、两篇量子 OS）。经典系统（内存/存储/网络/虚拟化/内核 + 形式化验证）合计 33 篇。
- **工业界参与（≥1 篇）：** Alibaba、Apple、Barkhausen Institut、ByteDance、Google、Huawei、Intel、Kernkonzept、Meta、Microsoft Research、NVIDIA、OpenAI —— 工业合作的广度明显超过"常见的两三家超大厂"。
- **本届首次显眼的主题：** 晶圆级 LLM 推理（[WaferLLM](../papers/osdi-2025/waferllm-large-language-model-inference-at-wafer-scale.md)）、两篇正面讨论量子 OS 的工作、以及一篇少见的串行性能优化方法论论文（[Principles and Methodologies](../papers/osdi-2025/principles-and-methodologies-for-serial-performance-optimization.md)）。
