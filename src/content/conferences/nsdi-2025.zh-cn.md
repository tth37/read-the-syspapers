NSDI '25 一共录用了 **83 篇**,分两个 session 排开。整届程序读下来,像是一个已经把 ML 负载的分量消化进来、但又没放弃自家议程的网络会议。大模型训练与推理基础设施如今自成一条完整 track——83 篇里有 14 篇直接落在集合通信、检查点、故障定位或推理服务上——连经典的数据中心传输 track,也越来越习惯拿阿里云 CDN、字节跳动训练集群、Google Colossus、Meta 联邦分析的部署数据说话。除此之外,还有两股贯穿全会的张力:一是「可编程」这个词如今已经扩到整栈——SmartNIC、Tofino 交换机、RNIC、eBPF、service mesh 在 NSDI '25 上都配齐了编译器和验证器;二是运行时诊断取代了离线分析,无论对象是 RDMA 容器集群、某个分布式协议的实现,还是 12k GPU 规模的训练作业,大家都更愿意让系统在线上自己说话。

## 主题

- **大模型与 ML 负载重塑了网络议程。** 训练基础设施这一块尤其热闹:[SimAI](../papers/nsdi-2025/simai-unifying-architecture-design-and-performance-tuning-for-large-scale-large-language.md) 把容量规划和包级调优统一到同一个仿真器里,和真实训练对齐到 `98.1%`;[RLHFuse](../papers/nsdi-2025/optimizing-rlhf-training-for-large-language-models-with-stage-fusion.md) 把 RLHF 的各阶段融合起来,吞吐拉高到 `3.7x`;[ByteCheckpoint](../papers/nsdi-2025/bytecheckpoint-a-unified-checkpointing-system-for-large-foundation-model-development.md) 做成加载时可再分片的统一检查点;[Multiverse](../papers/nsdi-2025/accelerating-design-space-exploration-for-llm-training-systems-with-multi-experiment.md) 则把多组仿真打包成 ECS 流水线,设计空间探索加速 `73x`。诊断方向自成一簇:[Holmes](../papers/nsdi-2025/holmes-localizing-irregularities-in-llm-training-with-mega-scale-gpu-clusters.md)、[Minder](../papers/nsdi-2025/minder-faulty-machine-detection-for-large-scale-distributed-model-training.md) 和 [Aegis](../papers/nsdi-2025/evolution-of-aegis-fault-diagnosis-for-ai-model-training-service-in-production.md) 都能几秒之内从 CCL 日志里揪出异常机器。集合通信这边,[AutoCCL](../papers/nsdi-2025/autoccl-automated-collective-communication-tuning-for-accelerating-distributed-and.md) 和 [OptiReduce](../papers/nsdi-2025/optireduce-resilient-and-tail-optimal-allreduce-for-distributed-deep-learning-in-the-cloud.md) 选择在线调优;而 [SuperServe](../papers/nsdi-2025/superserve-fine-grained-inference-serving-for-unpredictable-workloads.md)、[Prism](../papers/nsdi-2025/gpu-disaggregated-serving-for-deep-learning-recommendation-models-at-scale.md) 和 [RegenHance](../papers/nsdi-2025/region-based-content-enhancement-for-ef-cient-video-analytics-at-the-edge.md) 把推理、DLRM 与边缘分析一同推进到「细粒度、松弛感知」这同一片设计空间里。

- **可编程数据面进入了全栈时代。** [Alkali](../papers/nsdi-2025/enabling-portable-and-high-performance-smartnic-programs-with-alkali.md) 把单线程 SmartNIC 程序编译到异构 NIC 上,切分点和状态放置都是自动决定的;[CRAM](../papers/nsdi-2025/scaling-ip-lookup-to-large-databases-using-the-cram-lens.md) 把 Tofino-2 当成 TCAM+SRAM 的联合基底,IP 查找规模远远超出纯 TCAM 的方案。[State-Compute Replication](../papers/nsdi-2025/state-compute-replication-parallelizing-high-speed-stateful-packet-processing.md) 在多核间做报文 round-robin,并用 piggyback 的方式带上历史,让有状态的流也能跨核扩展;[ClubHeap](../papers/nsdi-2025/clubheap-a-high-speed-and-scalable-priority-queue-for-programmable-packet-scheduling.md)、[PACKS](../papers/nsdi-2025/everything-matters-in-programmable-packet-scheduling.md) 与 [SCRR](../papers/nsdi-2025/self-clocked-round-robin-packet-scheduling.md) 直接在真实硬件上把 PIFO 级的优先级调度做了出来。顺带一提,[P4RTC](../papers/nsdi-2025/when-p4-meets-run-to-completion-architecture.md) 把 P4 搬到了 run-to-completion ASIC 上;[OrbitCache](../papers/nsdi-2025/pushing-the-limits-of-in-network-caching-for-key-value-stores.md) 让热 KV 以报文的形式一直在交换机里循环;[CATO](../papers/nsdi-2025/cato-end-to-end-optimization-of-ml-based-traffic-analysis-pipelines.md) 则把特征选择和抓包深度放在一起做联合搜索,用于 ML 流量分析。

- **Far memory 与细粒度资源成了能真正铺下去的底座。** [Beehive](../papers/nsdi-2025/beehive-a-scalable-disaggregated-memory-runtime-exploiting-asynchrony-of-multithreaded.md) 把远端访问编译成轻量的 pararoutine,省掉了过去那套「一堆 uthread 加上下文切换」的写法;[Eden](../papers/nsdi-2025/eden-developer-friendly-application-integrated-far-memory.md) 只需要程序员给一点点提示,大多数 far-memory miss 就能在用户态处理掉。[ODRP](../papers/nsdi-2025/odrp-on-demand-remote-paging-with-programmable-rdma.md) 用链式 RDMA WR 把 RNIC 变成一块 zero-CPU 的远端 swap 设备,页粒度也做到了 `4 KB`;[Quicksand](../papers/nsdi-2025/quicksand-harnessing-stranded-datacenter-resources-with-granular-computing.md) 和 [GRANNY](../papers/nsdi-2025/granny-granular-management-of-compute-intensive-applications-in-the-cloud.md) 的思路一致——把任务切小、让它可迁移,从而把零散的 CPU 和 RAM 捡回来。到头来这件事的最后一公里是 [Leopard](../papers/nsdi-2025/making-serverless-pay-for-use-a-reality-with-leopard.md) 走完的:直接把 spot / preemptible 资源接进 Serverless 的计费模型。

- **部署故事回过头来说话。** 本届罕见地聚集了一批大厂论文,带来的都是真实测量结果:Google 用 [hotspot-aware placement](../papers/nsdi-2025/preventing-network-bottlenecks-accelerating-datacenter-services-with-hotspot-aware.md) 把 ToR 热点削掉了 `90%`,又公开了 [Aequitas 在存储 RPC 上做 QoS 对齐](../papers/nsdi-2025/learnings-from-deploying-network-qos-alignment-to-application-priorities-for-storage.md)的完整过程;阿里云一口气给出 [Aegis 训练诊断](../papers/nsdi-2025/evolution-of-aegis-fault-diagnosis-for-ai-model-training-service-in-production.md)、[CDN 拥塞控制选型](../papers/nsdi-2025/learning-production-optimized-congestion-control-selection-for-alibaba-cloud-cdn.md) 和 [RDMA 容器网络扩展性](../papers/nsdi-2025/mitigating-scalability-walls-of-rdma-based-container-networks.md) 三篇;字节跳动贡献了 [ByteCheckpoint](../papers/nsdi-2025/bytecheckpoint-a-unified-checkpointing-system-for-large-foundation-model-development.md) 与 [Minder](../papers/nsdi-2025/minder-faulty-machine-detection-for-large-scale-distributed-model-training.md);Meta 则详细讲了 [PAPAYA](../papers/nsdi-2025/papaya-federated-analytics-stack-engineering-privacy-scalability-and-practicality.md) 在 ~100M 手机上的联邦分析。与此同时,一篇关于 [fail-slow 容忍](../papers/nsdi-2025/one-size-fits-none-understanding-and-enhancing-slow-fault-tolerance-in-modern-distributed.md) 的跨系统研究用自适应检测顶掉了那些脆弱的固定超时。

- **形式化验证在各层之间铺了过去。** [NDD](../papers/nsdi-2025/ndd-a-decision-diagram-for-network-verification.md) 把 BDD 式的网络验证抬到字段层,去掉了原子爆炸这个老瓶颈;[Velo](../papers/nsdi-2025/verifying-maximum-link-loads-in-a-changing-world.md) 在故障和 BGP 变更下验证每条链路的最坏负载;[p4tv](../papers/nsdi-2025/on-temporal-verification-of-stateful-p4-programs.md) 用 P4LTL 把时序验证带进 stateful P4。[VEP](../papers/nsdi-2025/vep-a-two-stage-verification-toolchain-for-full-ebpf-programmability.md) 把 eBPF 的证明搜索挪到用户态,内核里只留一个很小的 checker;[QUASI](../papers/nsdi-2025/a-layered-formal-methods-approach-to-answering-queue-related-queries.md) 用一层可靠抽象配合 SMT 来回答队列相关查询;[MeshTest](../papers/nsdi-2025/meshtest-end-to-end-testing-for-service-mesh-traffic-management.md) 则直接为 service mesh 生成连通的配置和期望结果。其余几篇在运行时做验证:[Ellsberg](../papers/nsdi-2025/runtime-protocol-refinement-checking-for-distributed-protocol-implementations.md) 把分布式实现对着协议模型做 refinement check;[Smart Casual](../papers/nsdi-2025/smart-casual-verification-of-the-confidential-consortium-framework.md) 把 TLA+ 和 Microsoft CCF 的 C++ 测试通过 CI 里的 trace 验证串起来;[CEGS](../papers/nsdi-2025/cegs-configuration-example-generalizing-synthesizer.md) 则靠 GNN + LLM + SMT 的组合,从厂商示例配置里综合出新拓扑下该用的配置。

## 值得关注的趋势

- **让系统学会选,而不是手工调。** [Mutant](../papers/nsdi-2025/mutant-learning-congestion-control-from-existing-protocols-via-online-reinforcement.md) 用 contextual bandit、[ALI CCS](../papers/nsdi-2025/learning-production-optimized-congestion-control-selection-for-alibaba-cloud-cdn.md) 直接吃生产日志、[Mowgli](../papers/nsdi-2025/mowgli-passively-learned-rate-control-for-real-time-video.md) 则在 GCC 遥测上做保守的 offline RL——三篇都是在线决定用哪种速率控制,而不是在编译期把一种拥塞控制钉死。[BFTBrain](../papers/nsdi-2025/bftbrain-adaptive-bft-consensus-with-reinforcement-learning.md) 把同一招搬到了 Byzantine 协议上;[AutoCCL](../papers/nsdi-2025/autoccl-automated-collective-communication-tuning-for-accelerating-distributed-and.md) 与 [CATO](../papers/nsdi-2025/cato-end-to-end-optimization-of-ml-based-traffic-analysis-pipelines.md) 则用贝叶斯优化去搜索调参或特征维度。说到底,共同的一步是:把旋钮空间当成一等对象,再把搜索过程扔到线上。

- **传输被拆成三件事:谁调度、谁暂停、谁反馈。** [SIRD](../papers/nsdi-2025/sird-a-sender-informed-receiver-driven-datacenter-transport-protocol.md) 在独占下行链路上延续 receiver-driven 的调度,共享链路上则靠 sender-informed 的 credit rationing;[Pyrrha](../papers/nsdi-2025/pyrrha-congestion-root-based-flow-control-to-eliminate-head-of-line-blocking-in-datacenter.md) 只去暂停汇入同一 congestion root 的流;[MTP](../papers/nsdi-2025/mtp-transport-for-in-network-computing.md) 把传输迁到 message 和 pathlet 这一层,让网内 offload 能安全地改写流量。[White-Boxing RDMA](../papers/nsdi-2025/white-boxing-rdma-with-packet-granular-software-control.md) 把 BlueField-3 的 per-QP 出队变成软件能控的点;[eTran](../papers/nsdi-2025/etran-extensible-kernel-transport-with-ebpf.md) 则靠 eBPF 钩子,让内核传输的性能向用户态看齐。

- **运行时诊断顶掉了离线分析。** [Ellsberg](../papers/nsdi-2025/runtime-protocol-refinement-checking-for-distributed-protocol-implementations.md) 与 [Smart Casual](../papers/nsdi-2025/smart-casual-verification-of-the-confidential-consortium-framework.md) 用 trace 把已部署的代码和形式模型对齐;[Holmes](../papers/nsdi-2025/holmes-localizing-irregularities-in-llm-training-with-mega-scale-gpu-clusters.md)、[Minder](../papers/nsdi-2025/minder-faulty-machine-detection-for-large-scale-distributed-model-training.md) 与 [Aegis](../papers/nsdi-2025/evolution-of-aegis-fault-diagnosis-for-ai-model-training-service-in-production.md) 从 CCL 日志里定位异常机器;[ntprof](../papers/nsdi-2025/understanding-and-profiling-nvme-over-tcp-using-ntprof.md) 把 NVMe/TCP 看作一条软件 switch 链来剖析;[ScalaCN](../papers/nsdi-2025/mitigating-scalability-walls-of-rdma-based-container-networks.md) 直接在真实的 RDMA 容器网络上跑 combinatorial causal testing;而 [One-Size-Fits-None](../papers/nsdi-2025/one-size-fits-none-understanding-and-enhancing-slow-fault-tolerance-in-modern-distributed.md) 则用运行时自适应的 slow-fault 检测,把固定阈值替换掉。

- **可编程交换机终于不只做匹配了,也来做调度、优先级与缓存。** [ClubHeap](../papers/nsdi-2025/clubheap-a-high-speed-and-scalable-priority-queue-for-programmable-packet-scheduling.md) 把 PIFO 完整流水化,做到每个 cycle 一次 replace;[PACKS](../papers/nsdi-2025/everything-matters-in-programmable-packet-scheduling.md) 把分位数准入与占用感知的队列映射组合起来;[SCRR](../papers/nsdi-2025/self-clocked-round-robin-packet-scheduling.md) 则用 self-clocked 虚拟时间替掉 DRR 的固定 quantum。[OrbitCache](../papers/nsdi-2025/pushing-the-limits-of-in-network-caching-for-key-value-stores.md) 把变长 KV 做成网内循环的报文来缓存;[CRAM](../papers/nsdi-2025/scaling-ip-lookup-to-large-databases-using-the-cram-lens.md) 把现代 IP 查找表塞进了 Tofino-2;[P4RTC](../papers/nsdi-2025/when-p4-meets-run-to-completion-architecture.md) 则把 run-to-completion ASIC 做成一等的 P4 目标。

## 必读推荐

- **[SimAI](../papers/nsdi-2025/simai-unifying-architecture-design-and-performance-tuning-for-large-scale-large-language.md)** —— 难得一见的 LLM 训练仿真器,直接复用真实训练框架和 NCCL,把容量规划与包级调优拼到同一条流水上,和真实训练的对齐度做到了 `98.1%`。这种工具可以直接拿来做大规模训练架构上线前的决策。
- **[Beehive](../papers/nsdi-2025/beehive-a-scalable-disaggregated-memory-runtime-exploiting-asynchrony-of-multithreaded.md)** —— 用 pararoutine 取代 far memory 一贯依赖的「多 uthread + 上下文切换」,让解聚合内存运行时在源码层继续保持同步代码的样子。
- **[Ellsberg](../papers/nsdi-2025/runtime-protocol-refinement-checking-for-distributed-protocol-implementations.md)** —— 提出了运行时的协议 refinement checking:实现照常部署,从运行 trace 里重建协议行为,再拿去对照模型;既不改服务,也不做跨节点协调。
- **[Aequitas](../papers/nsdi-2025/learnings-from-deploying-network-qos-alignment-to-application-priorities-for-storage.md)** —— 来自 Google 的生产证据:只要把 RPC 到 DSCP/WFQ 的对齐修好,存储延迟就能降下来,即便为此要牺牲一部分流的优先级也值得。这种部署报告正是 NSDI 最该承载的那类工作。
- **[Smart Casual Verification of CCF](../papers/nsdi-2025/smart-casual-verification-of-the-confidential-consortium-framework.md)** —— 把 TLA+ 和 CCF 的 C++ 测试在 CI 里以 trace validation 的方式对起来,为一个真实运行的 confidential-consortium 服务,打通了一条从规范到实现的可复现路径,其他生产服务完全可以照着做。

## 数据概览

- **已综述论文数:** 83(两个 session 的完整程序)。
- **分类分布:** 数据中心网络与传输 19、大模型与 ML 训练/推理 14、内存/Serverless/存储 12、可编程交换机与智能报文处理 10、网络验证与综合 9、无线/蜂窝/实时媒体 8、安全与隐私 6、共识与区块链 5。
