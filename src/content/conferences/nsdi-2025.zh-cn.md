NSDI '25 共收录 **83 篇论文**，分两个 session 呈现。整届程序读起来像一个已经把 ML workload 压力消化进来、但又不放弃自己版图的网络会议。大模型训练与推理基础设施成了一条完整的 track——83 篇里有 14 篇直接讨论集合通信、检查点、故障定位或推理服务——即便是经典的数据中心传输 track，也普遍引用来自阿里云 CDN、字节跳动训练集群、Google Colossus 与 Meta 联邦分析的部署证据。两股贯穿全会的张力组织了其余内容：**"可编程"在这一届扩展到全栈**（SmartNIC、Tofino 交换机、RNIC、eBPF、service mesh 在 NSDI '25 上都配齐了编译器与验证器），以及**运行时诊断取代离线分析**成为理解分布式行为的首选，无论目标是 RDMA 容器集群、分布式协议实现，还是 12k GPU 规模的训练作业。

## 主题

- **大模型与 ML 负载重塑网络议程。** 训练基础设施囊括：[SimAI](../papers/nsdi-2025/simai-unifying-architecture-design-and-performance-tuning-for-large-scale-large-language.md) 把容量规划与包级调优统一到与真实训练 98.1% 对齐的仿真器上、[RLHFuse](../papers/nsdi-2025/optimizing-rlhf-training-for-large-language-models-with-stage-fusion.md) 融合 RLHF 各阶段带来 3.7x 吞吐、[ByteCheckpoint](../papers/nsdi-2025/bytecheckpoint-a-unified-checkpointing-system-for-large-foundation-model-development.md) 在加载时就支持再分片的统一检查点，以及 [Multiverse](../papers/nsdi-2025/accelerating-design-space-exploration-for-llm-training-systems-with-multi-experiment.md) 把多组训练仿真打包成 ECS 流水、带来 73x 的设计空间探索加速。诊断自成一簇：[Holmes](../papers/nsdi-2025/holmes-localizing-irregularities-in-llm-training-with-mega-scale-gpu-clusters.md)、[Minder](../papers/nsdi-2025/minder-faulty-machine-detection-for-large-scale-distributed-model-training.md) 与 [Aegis](../papers/nsdi-2025/evolution-of-aegis-fault-diagnosis-for-ai-model-training-service-in-production.md) 都能在几秒内从 CCL 日志里定位异常机器。集合通信则由 [AutoCCL](../papers/nsdi-2025/autoccl-automated-collective-communication-tuning-for-accelerating-distributed-and.md) 与 [OptiReduce](../papers/nsdi-2025/optireduce-resilient-and-tail-optimal-allreduce-for-distributed-deep-learning-in-the-cloud.md) 在线调优，而 [SuperServe](../papers/nsdi-2025/superserve-fine-grained-inference-serving-for-unpredictable-workloads.md)、[Prism](../papers/nsdi-2025/gpu-disaggregated-serving-for-deep-learning-recommendation-models-at-scale.md) 与 [RegenHance](../papers/nsdi-2025/region-based-content-enhancement-for-ef-cient-video-analytics-at-the-edge.md) 把推理、DLRM 与边缘分析推进同一个"细粒度 + 松弛感知"的设计空间。

- **可编程数据面进入"全栈"时代。** [Alkali](../papers/nsdi-2025/enabling-portable-and-high-performance-smartnic-programs-with-alkali.md) 把单线程 SmartNIC 程序自动切分、复制并做状态安置，编译到异构 NIC；[CRAM](../papers/nsdi-2025/scaling-ip-lookup-to-large-databases-using-the-cram-lens.md) 把 Tofino-2 视为 TCAM+SRAM 联合基底，把 IP 查找表规模远远推过纯 TCAM 方案；[State-Compute Replication](../papers/nsdi-2025/state-compute-replication-parallelizing-high-speed-stateful-packet-processing.md) 在核间做包级 round-robin，并 piggyback 有界 packet history，让单条有状态流也能跨核扩展；[ClubHeap](../papers/nsdi-2025/clubheap-a-high-speed-and-scalable-priority-queue-for-programmable-packet-scheduling.md)、[PACKS](../papers/nsdi-2025/everything-matters-in-programmable-packet-scheduling.md) 与 [SCRR](../papers/nsdi-2025/self-clocked-round-robin-packet-scheduling.md) 在真实硬件上做出 PIFO 级的优先级调度；[P4RTC](../papers/nsdi-2025/when-p4-meets-run-to-completion-architecture.md) 把 P4 迁移到 run-to-completion ASIC；[OrbitCache](../papers/nsdi-2025/pushing-the-limits-of-in-network-caching-for-key-value-stores.md) 让热 KV 以报文形式在交换机内环绕；[CATO](../papers/nsdi-2025/cato-end-to-end-optimization-of-ml-based-traffic-analysis-pipelines.md) 联合搜索特征与抓包深度，优化 ML 流量分析。

- **Far memory 与细粒度资源成为可部署基底。** [Beehive](../papers/nsdi-2025/beehive-a-scalable-disaggregated-memory-runtime-exploiting-asynchrony-of-multithreaded.md) 把远端访问编译成轻量 pararoutine，代替"多 uthread + 上下文切换"套路；[Eden](../papers/nsdi-2025/eden-developer-friendly-application-integrated-far-memory.md) 在少量程序员提示下让 far-memory miss 大多在用户态完成；[ODRP](../papers/nsdi-2025/odrp-on-demand-remote-paging-with-programmable-rdma.md) 用链式 RDMA WR 把 RNIC 变成 zero-CPU 的远端 swap 设备并支持 4 KB 页级分配；[Quicksand](../papers/nsdi-2025/quicksand-harnessing-stranded-datacenter-resources-with-granular-computing.md) 与 [GRANNY](../papers/nsdi-2025/granny-granular-management-of-compute-intensive-applications-in-the-cloud.md) 把作业切得小而可迁移，回收散落的 CPU/RAM；[Leopard](../papers/nsdi-2025/making-serverless-pay-for-use-a-reality-with-leopard.md) 直接把 spot / preemptible 资源暴露到 Serverless 的计费模型里，把"按使用付费"推到最后一步。

- **部署故事密集反馈。** 本届难得地集中了一批大厂论文：Google 用 [hotspot-aware placement](../papers/nsdi-2025/preventing-network-bottlenecks-accelerating-datacenter-services-with-hotspot-aware.md) 把 ToR 热点降低 90%，并在 [Aequitas 的存储 RPC QoS 对齐](../papers/nsdi-2025/learnings-from-deploying-network-qos-alignment-to-application-priorities-for-storage.md)上给出了完整的部署记录；阿里云贡献了 [Aegis 训练诊断](../papers/nsdi-2025/evolution-of-aegis-fault-diagnosis-for-ai-model-training-service-in-production.md)、[CDN 拥塞控制选型](../papers/nsdi-2025/learning-production-optimized-congestion-control-selection-for-alibaba-cloud-cdn.md) 与 [RDMA 容器网络扩展性](../papers/nsdi-2025/mitigating-scalability-walls-of-rdma-based-container-networks.md)；字节跳动带来了 [ByteCheckpoint](../papers/nsdi-2025/bytecheckpoint-a-unified-checkpointing-system-for-large-foundation-model-development.md) 与 [Minder](../papers/nsdi-2025/minder-faulty-machine-detection-for-large-scale-distributed-model-training.md)；Meta 详述 [PAPAYA](../papers/nsdi-2025/papaya-federated-analytics-stack-engineering-privacy-scalability-and-practicality.md) 在接近 1 亿手机上的联邦分析；而关于 [fail-slow 容忍](../papers/nsdi-2025/one-size-fits-none-understanding-and-enhancing-slow-fault-tolerance-in-modern-distributed.md) 的跨系统研究则用运行时自适应检测取代了脆弱的固定超时。

- **形式化验证跨栈延伸。** [NDD](../papers/nsdi-2025/ndd-a-decision-diagram-for-network-verification.md) 把 BDD 式网络验证抬升到字段级，消除原子爆炸瓶颈；[Velo](../papers/nsdi-2025/verifying-maximum-link-loads-in-a-changing-world.md) 在故障与 BGP 变化下验证每条链路的最坏负载；[p4tv](../papers/nsdi-2025/on-temporal-verification-of-stateful-p4-programs.md) 用 P4LTL 把时序验证带入 stateful P4；[VEP](../papers/nsdi-2025/vep-a-two-stage-verification-toolchain-for-full-ebpf-programmability.md) 把 eBPF 证明搜索迁移到用户态 C，并在内核只保留很小的 checker；[QUASI](../papers/nsdi-2025/a-layered-formal-methods-approach-to-answering-queue-related-queries.md) 用可靠抽象层配合 SMT 处理队列相关查询；[MeshTest](../papers/nsdi-2025/meshtest-end-to-end-testing-for-service-mesh-traffic-management.md) 为 service mesh 生成连通配置与期望结果；[Ellsberg](../papers/nsdi-2025/runtime-protocol-refinement-checking-for-distributed-protocol-implementations.md) 在运行时对分布式实现做 protocol refinement check；[Smart Casual](../papers/nsdi-2025/smart-casual-verification-of-the-confidential-consortium-framework.md) 把 TLA+ 与 CCF 的 C++ 测试在 CI 里做 trace 校验对齐；[CEGS](../papers/nsdi-2025/cegs-configuration-example-generalizing-synthesizer.md) 用 GNN + LLM + SMT 流水从厂商示例配置中综合目标拓扑下的正确配置。

## 值得关注的趋势

- **学出"用哪个"，而不是手工调参。** [Mutant](../papers/nsdi-2025/mutant-learning-congestion-control-from-existing-protocols-via-online-reinforcement.md)、[ALI CCS](../papers/nsdi-2025/learning-production-optimized-congestion-control-selection-for-alibaba-cloud-cdn.md) 与 [Mowgli](../papers/nsdi-2025/mowgli-passively-learned-rate-control-for-real-time-video.md) 分别用 contextual bandit、生产日志和在 GCC 遥测上做的保守 offline RL，在线选择拥塞控制算法，而不是在编译期固定一个。[BFTBrain](../papers/nsdi-2025/bftbrain-adaptive-bft-consensus-with-reinforcement-learning.md) 把同样思路用在 BFT 协议上；[AutoCCL](../papers/nsdi-2025/autoccl-automated-collective-communication-tuning-for-accelerating-distributed-and.md) 与 [CATO](../papers/nsdi-2025/cato-end-to-end-optimization-of-ml-based-traffic-analysis-pipelines.md) 则用贝叶斯优化搜索调优或特征维度。共同动作是：把旋钮空间当一等对象，并把搜索放到线上。

- **传输被拆解：谁调度、谁暂停、谁反馈。** [SIRD](../papers/nsdi-2025/sird-a-sender-informed-receiver-driven-datacenter-transport-protocol.md) 在独占下行链路上继续用 receiver-driven 调度，但在共享链路上靠 sender-informed 的 credit rationing；[Pyrrha](../papers/nsdi-2025/pyrrha-congestion-root-based-flow-control-to-eliminate-head-of-line-blocking-in-datacenter.md) 只暂停汇入每个 congestion root 的流；[MTP](../papers/nsdi-2025/mtp-transport-for-in-network-computing.md) 把传输迁移到 message 与 pathlet，让网内 offload 可以安全地改写流量；[White-Boxing RDMA](../papers/nsdi-2025/white-boxing-rdma-with-packet-granular-software-control.md) 把 BlueField-3 的 per-QP 出队速率变成软件控制点；[eTran](../papers/nsdi-2025/etran-extensible-kernel-transport-with-ebpf.md) 通过 eBPF 钩子让内核传输接近用户态的性能。

- **运行时诊断取代离线分析。** [Ellsberg](../papers/nsdi-2025/runtime-protocol-refinement-checking-for-distributed-protocol-implementations.md) 与 [Smart Casual](../papers/nsdi-2025/smart-casual-verification-of-the-confidential-consortium-framework.md) 在运行时用 trace 对齐形式模型与部署实现；[Holmes](../papers/nsdi-2025/holmes-localizing-irregularities-in-llm-training-with-mega-scale-gpu-clusters.md)、[Minder](../papers/nsdi-2025/minder-faulty-machine-detection-for-large-scale-distributed-model-training.md) 与 [Aegis](../papers/nsdi-2025/evolution-of-aegis-fault-diagnosis-for-ai-model-training-service-in-production.md) 从 CCL 日志里定位异常机器；[ntprof](../papers/nsdi-2025/understanding-and-profiling-nvme-over-tcp-using-ntprof.md) 把 NVMe/TCP 当作软件 switch 链来剖析；[ScalaCN](../papers/nsdi-2025/mitigating-scalability-walls-of-rdma-based-container-networks.md) 在真实 RDMA 容器网络上做 combinatorial causal testing；[One-Size-Fits-None](../papers/nsdi-2025/one-size-fits-none-understanding-and-enhancing-slow-fault-tolerance-in-modern-distributed.md) 用运行时自适应的 slow-fault 检测替换固定阈值。

- **可编程交换机终于能做调度、优先级与缓存，不止匹配。** [ClubHeap](../papers/nsdi-2025/clubheap-a-high-speed-and-scalable-priority-queue-for-programmable-packet-scheduling.md) 把 PIFO 完整流水化，做到每 cycle 一次 replace；[PACKS](../papers/nsdi-2025/everything-matters-in-programmable-packet-scheduling.md) 把分位数准入与占用感知队列映射结合起来；[SCRR](../papers/nsdi-2025/self-clocked-round-robin-packet-scheduling.md) 用 self-clocked 虚拟时间代替 DRR 的固定 quantum；[OrbitCache](../papers/nsdi-2025/pushing-the-limits-of-in-network-caching-for-key-value-stores.md) 把变长 KV 缓存做成交换机内循环的报文；[CRAM](../papers/nsdi-2025/scaling-ip-lookup-to-large-databases-using-the-cram-lens.md) 把现代 IP 查找表装进 Tofino-2；[P4RTC](../papers/nsdi-2025/when-p4-meets-run-to-completion-architecture.md) 把 run-to-completion ASIC 做成一等的 P4 目标。

## 必读推荐

- **[SimAI](../papers/nsdi-2025/simai-unifying-architecture-design-and-performance-tuning-for-large-scale-large-language.md)** —— 难得一见的 LLM 训练仿真器：复用真实训练框架与 NCCL，把容量规划与包级调优统一到一条与真实训练 98.1% 对齐的仿真管线上，可以直接用于大规模训练架构的上线前决策。
- **[Beehive](../papers/nsdi-2025/beehive-a-scalable-disaggregated-memory-runtime-exploiting-asynchrony-of-multithreaded.md)** —— 用 pararoutine 取代 far memory 一贯的"多 uthread + 上下文切换"套路，让解聚合内存运行时在源码层面保持同步风格的编程模型。
- **[Ellsberg](../papers/nsdi-2025/runtime-protocol-refinement-checking-for-distributed-protocol-implementations.md)** —— 提出运行时协议 refinement checking：原样部署、从消息 trace 重建协议行为、再对齐到协议模型；不用改服务、也不用跨节点协调。
- **[Aequitas](../papers/nsdi-2025/learnings-from-deploying-network-qos-alignment-to-application-priorities-for-storage.md)** —— Google 规模的生产证据：修正 RPC 到 DSCP/WFQ 的对齐可以降低存储延迟，即便伴随对部分流的降级也如此。这类部署报告正是 NSDI 最适合承载的工作。
- **[Smart Casual Verification of CCF](../papers/nsdi-2025/smart-casual-verification-of-the-confidential-consortium-framework.md)** —— 把 TLA+ 与 CCF 的 C++ 测试在 CI 里做 trace validation，为一个真实运行的 confidential-consortium 服务建立了"从规范到实现"的可复现路径。可被其他生产服务直接借鉴。

## 数据概览

- **已综述论文数：** 83（覆盖两个 session 的完整程序）。
- **分类分布：** 数据中心网络与传输 19，大模型与 ML 训练/推理 14，内存/Serverless/存储 12，可编程交换机与智能报文处理 10，网络验证与综合 9，无线/蜂窝/实时媒体 8,安全与隐私 6，共识与区块链 5。
- **ML vs. 经典系统：** llm-and-ml-training-serving 约 14 篇（约 17%）；另有约 6 篇在其他 track 带 ML 成分（[CATO](../papers/nsdi-2025/cato-end-to-end-optimization-of-ml-based-traffic-analysis-pipelines.md)、[ZOOMSYNTH](../papers/nsdi-2025/resolving-packets-from-counters-enabling-multi-scale-network-traffic-super-resolution-via.md)、[Mutant](../papers/nsdi-2025/mutant-learning-congestion-control-from-existing-protocols-via-online-reinforcement.md)、[ALI CCS](../papers/nsdi-2025/learning-production-optimized-congestion-control-selection-for-alibaba-cloud-cdn.md)、[Mowgli](../papers/nsdi-2025/mowgli-passively-learned-rate-control-for-real-time-video.md)、[BFTBrain](../papers/nsdi-2025/bftbrain-adaptive-bft-consensus-with-reinforcement-learning.md)），剩下约 63 篇仍属经典网络/系统工作。
- **带生产部署的论文（≥1 位作者来自运营方）：** Google（[hotspot-aware placement](../papers/nsdi-2025/preventing-network-bottlenecks-accelerating-datacenter-services-with-hotspot-aware.md)、[Aequitas](../papers/nsdi-2025/learnings-from-deploying-network-qos-alignment-to-application-priorities-for-storage.md)）、阿里云（[Aegis](../papers/nsdi-2025/evolution-of-aegis-fault-diagnosis-for-ai-model-training-service-in-production.md)、[ALI CCS](../papers/nsdi-2025/learning-production-optimized-congestion-control-selection-for-alibaba-cloud-cdn.md)、[ScalaCN](../papers/nsdi-2025/mitigating-scalability-walls-of-rdma-based-container-networks.md)、[SimAI](../papers/nsdi-2025/simai-unifying-architecture-design-and-performance-tuning-for-large-scale-large-language.md)）、字节跳动（[ByteCheckpoint](../papers/nsdi-2025/bytecheckpoint-a-unified-checkpointing-system-for-large-foundation-model-development.md)、[Minder](../papers/nsdi-2025/minder-faulty-machine-detection-for-large-scale-distributed-model-training.md)）、Meta（[PAPAYA](../papers/nsdi-2025/papaya-federated-analytics-stack-engineering-privacy-scalability-and-practicality.md)）、Microsoft Azure Research（[Smart Casual](../papers/nsdi-2025/smart-casual-verification-of-the-confidential-consortium-framework.md)）。工业参与比通常的 NSDI 年份明显更广。
- **本届首次显眼的主题：** LLM 训练诊断已自成子方向（Holmes、Minder、Aegis 三篇都针对 10k GPU 级集群）、可编程交换机的优先级调度（ClubHeap、PACKS、SCRR 同一 session 三篇）、运行时协议 refinement checking（[Ellsberg](../papers/nsdi-2025/runtime-protocol-refinement-checking-for-distributed-protocol-implementations.md)），以及细粒度 Serverless 计费（[Leopard](../papers/nsdi-2025/making-serverless-pay-for-use-a-reality-with-leopard.md)）。
