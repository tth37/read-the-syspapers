SOSP '25 在首尔共录用 **66 篇**论文。ML 系统已经从少数派变成第一大类:66 篇里有 17 篇直接做 LLM serving 或训练基础设施,另有 6 篇 GPU/加速器底层工作几乎都因为 ML 才存在。经典 SOSP 主线 — 内核、可验证系统、内存与存储、分布式数据库、调度 — 仍然完整,但程序的重心已经明显转向 *为 ML workload 虚拟化 GPU 与远端内存的 substrate*。另一条更安静的线索贯穿全程:形式验证正变成日常工具而非英雄式工程,出现在内核、eBPF 扩展、hypervisor、训练计划、文件系统和 WASI 运行时里。今年程序的两组核心张力因此是 ML 侧的 **substrate vs. serving trick**,以及可靠性侧的 **一次性证明 vs. 运行时检查**。

## 主题

- **LLM serving 与 GPU substrate 在收敛到一组共享原语。**[Aegaeon](../papers/sosp-2025/aegaeon-effective-gpu-pooling-for-concurrent-llm-serving-on-the-market.md) 在 token 边界做自动伸缩,把 marketplace 模型池化到共享 GPU;[Jenga](../papers/sosp-2025/jenga-effective-memory-management-for-serving-llm-with-heterogeneity.md) 与 [DiffKV](../papers/sosp-2025/diffkv-differentiated-memory-management-for-large-language-models-with-parallel-kv.md) 为异构模型与 K/V 不对称敏感性重新设计 KV 内存;[Pie](../papers/sosp-2025/pie-a-programmable-serving-system-for-emerging-llm-applications.md) 把 KV pages 与生成步骤暴露成 inferlet 编程模型;[PrefillOnly](../papers/sosp-2025/prefillonly-an-inference-engine-for-prefill-only-workloads-in-large-language-model.md) 专门服务单 token 的 prefill 请求;[LithOS](../papers/sosp-2025/lithos-an-operating-system-for-efficient-machine-learning-on-gpus.md) 把整个控制面下放到 CUDA driver 边界以下,做 TPC 粒度调度与 kernel atomization。它们一起在论证:serving 系统现在需要的是 OS 式 substrate,而不是临时 scheduler。

- **检索、动态性与异构硬件正在把 LLM 推出同构 GPU 模型。**[HedraRAG](../papers/sosp-2025/hedrarag-co-optimizing-generation-and-retrieval-for-heterogeneous-rag-workflows.md) 与 [METIS](../papers/sosp-2025/metis-fast-quality-aware-rag-systems-with-configuration-adaptation.md) 协同优化检索与生成;[IC-Cache](../papers/sosp-2025/ic-cache-efficient-large-language-model-serving-via-in-context-caching.md) 把历史 request-response 对当作 in-context demonstration 在小/大模型间路由;[KTransformers](../papers/sosp-2025/ktransformers-unleashing-the-full-potential-of-cpu-gpu-hybrid-inference-for-moe-models.md) 在 CPU AMX 与 GPU 之间跑大型 MoE 推理;[HeteroInfer](../papers/sosp-2025/characterizing-mobile-soc-for-accelerating-heterogeneous-llm-inference.md) 把 LLM 工作切到移动端 GPU 与 NPU;[Tempo](../papers/sosp-2025/tempo-compiled-dynamic-deep-learning-with-symbolic-dependence-graphs.md) 把时间作为显式 tensor 维度,服务动态 LLM/RL workload。

- **训练基础设施已经独立成形为一类系统工作。**[ByteRobust](../papers/sosp-2025/robust-llm-training-infrastructure-at-bytedance.md) 记录了一个万 GPU 生产栈 isolation-first 的容错策略;[Mycroft](../papers/sosp-2025/mycroft-tracing-dependencies-in-collective-communication-towards-reliable-llm-training.md) 通过追踪集合通信依赖在秒级定位 hang;[Sailor](../papers/sosp-2025/sailor-automating-distributed-training-over-dynamic-heterogeneous-and-geo-distributed.md) 在资源动态变化下联合选择 allocation、3D 并行与 zone 放置;[DCP](../papers/sosp-2025/dcp-addressing-input-dynamism-in-long-context-training-via-dynamic-context-parallelism.md) 用 per-batch 的 block 放置取代固定 context-parallel 布局;[TrainVerify](../papers/sosp-2025/trainverify-equivalence-based-verification-for-distributed-llm-training.md) 证明分布式训练计划与逻辑模型等价。可靠性、验证与调度在这里汇合。

- **形式验证从工具箱里走出,走进了内核、verifier 与 hypervisor。**[Atmosphere](../papers/sosp-2025/atmosphere-practical-verified-kernels-with-rust-and-verus.md) 用 Rust+Verus 加上 raw-pointer 代码与 flat permission map 证明了一个完整功能微内核;[BCF](../papers/sosp-2025/prove-it-to-the-kernel-precise-extension-analysis-via-proof-guided-abstraction-refinement.md) 把 eBPF 难证的 refinement 证明丢到用户态、再线性时间检查,救回了 403 个被错误拒绝的程序;[TickTock](../papers/sosp-2025/ticktock-verified-isolation-in-a-production-embedded-os.md) 用 Flux 验证 Tock 的细粒度 MPU 进程隔离;[CortenMM](../papers/sosp-2025/cortenmm-efficient-memory-management-with-strong-correctness-guarantees.md) 验证了 range-transactional 内存管理的锁定核心;[Ghost in the Android Shell](../papers/sosp-2025/ghost-in-the-android-shell-pragmatic-test-oracle-specification-of-a-production-hypervisor.md) 把可执行 ghost-state 规范嵌入 pKVM;[AutoMan](../papers/sosp-2025/automan-facilitating-verified-distributed-systems-development-through-automatic-code.md) 把 Dafny actions 转成验证过的代码,只让开发者动手优化热路径。

- **内存仍在向多层、远端池与依赖资源解构。**[Scalable Far Memory](../papers/sosp-2025/scalable-far-memory-balancing-faults-and-evictions.md) 把 eviction 完全从 fault 中解耦;[Spirit](../papers/sosp-2025/spirit-fair-allocation-of-interdependent-resources-in-remote-memory-systems.md) 联合定价本地缓存与远端带宽;[Demeter](../papers/sosp-2025/demeter-a-scalable-and-elastic-tiered-memory-solution-for-virtualized-cloud-via-guest.md) 把 VM 分层下放到 guest,用 guest 驱动的 PEBS;[cache_ext](../papers/sosp-2025/cache-ext-customizing-the-page-cache-with-ebpf.md) 把 per-cgroup 替换策略放到一个 eBPF 接口背后;[How to Copy Memory](../papers/sosp-2025/how-to-copy-memory-coordinated-asynchronous-copy-as-a-first-class-os-service.md) 把内存拷贝做成一个 OS 管理的异步服务;[Scalable Address Spaces](../papers/sosp-2025/scalable-address-spaces-using-concurrent-interval-skiplist.md) 用并发区间 skiplist 取代粗粒度 mmap 锁。

## 显著趋势

- **把策略点放到 vendor driver 或 framework 之下。**LLM 与 ML 系统反复把控制面塞到一个固化 API 的下方:[LithOS](../papers/sosp-2025/lithos-an-operating-system-for-efficient-machine-learning-on-gpus.md) 在 CUDA driver API 上做 interpose;[PhoenixOS](../papers/sosp-2025/phoenixos-concurrent-os-level-gpu-checkpoint-and-restore-with-validated-speculation.md) 用 speculate-then-validate 的访问集做 OS 级并发 GPU C/R;[Mercury](../papers/sosp-2025/mercury-unlocking-multi-gpu-operator-optimization-for-llms-via-remote-memory-scheduling.md) 把 GPU 内存当作可调度存储;[GoFS](../papers/sosp-2025/managing-scalable-direct-storage-accesses-for-gpus-with-gofs.md) 把 F2FS 元数据与 NVMe 队列控制搬到 GPU 上;[cache_ext](../papers/sosp-2025/cache-ext-customizing-the-page-cache-with-ebpf.md) 与 [FlexGuard](../papers/sosp-2025/flexguard-fast-mutual-exclusion-independent-of-subscription.md) 把 eBPF hook 变成可定制的内核策略点。共同动作是:不 fork framework、不 fork vendor stack,但仍然拿到控制权。

- **预测 common case,在 rare case 上验证并回滚。**[Tiga](../papers/sosp-2025/tiga-accelerating-geo-distributed-transactions-with-synchronized-clocks.md) 为 geo 事务排序预测一个未来时间戳,再用 super-quorum log 校验;[Pesto](../papers/sosp-2025/pesto-cooking-up-high-performance-bft-queries.md) 对无冲突 BFT SQL 跳过全序;[PhoenixOS](../papers/sosp-2025/phoenixos-concurrent-os-level-gpu-checkpoint-and-restore-with-validated-speculation.md) 从 launch 参数推测 GPU kernel 的访问集;[Optimistic Recovery (Phoenix)](../papers/sosp-2025/optimistic-recovery-for-high-availability-software-via-partial-process-state-preservation.md) 主进程重启时复用部分长寿命状态;[Orthrus](../papers/sosp-2025/orthrus-efficient-and-timely-detection-of-silent-user-data-corruption-in-the-cloud-with.md) 在其它核上验证数据路径闭包。每篇都用很小的回滚/复核成本换 common case 的大幅收益。

- **缩小编程模型而不是扩大它。**[Dandelion](../papers/sosp-2025/unlocking-true-elasticity-for-the-cloud-native-era-with-dandelion.md) 用纯计算函数 DAG 取代 POSIX serverless sandbox;[Quilt](../papers/sosp-2025/quilt-resource-aware-merging-of-serverless-workflows.md) 把可获利的 serverless 子图合并成单个 LLVM 级进程;[µFork](../papers/sosp-2025/fork-supporting-posix-fork-within-a-single-address-space-os.md) 在单 CHERI 地址空间内模拟 POSIX fork;[Pie](../papers/sosp-2025/pie-a-programmable-serving-system-for-emerging-llm-applications.md) 用 inferlet 取代固定的 LLM serving loop。每篇都缩小了 API,让 runtime 能更激进地特化。

- **从已有 artifact 推导难写的东西。**[KNighter](../papers/sosp-2025/knighter-transforming-static-analysis-with-llm-synthesized-checkers.md) 从 bug-fix patch 用 LLM triage 合成 Clang Static Analyzer checker;[Fawkes](../papers/sosp-2025/fawkes-finding-data-durability-bugs-in-dbmss-via-recovered-data-state-verification.md) 用 DBMS trace 构造 checkpoint-rectified 状态图;[WASIT](../papers/sosp-2025/wasit-deep-and-continuous-differential-testing-of-webassembly-system-interface.md) 把 WASI 规范变成可执行约束与抽象资源状态;[Veritas](../papers/sosp-2025/ebpf-misbehavior-detection-fuzzing-with-a-specification-based-oracle.md) 把 eBPF fuzzing 与 Dafny 规范 oracle 配对。invariant 与 oracle 来自已存在的代码与规范,而非新写的手工版本。

## 必读推荐

- **[LithOS](../papers/sosp-2025/lithos-an-operating-system-for-efficient-machine-learning-on-gpus.md)** — 把 GPU OS 工作 framing 成 substrate 而非更多 point scheduler:TPC 粒度调度、透明 kernel atomization、right-sizing 与 DVFS 在 CUDA driver 边界下被统一起来。
- **[Aegaeon](../papers/sosp-2025/aegaeon-effective-gpu-pooling-for-concurrent-llm-serving-on-the-market.md)** — token 边界自动伸缩在模型 marketplace 上最清晰的实证:阿里云 beta 部署节省 82% GPU,可承受 arrival rate 提升 2-2.5×。
- **[Atmosphere](../papers/sosp-2025/atmosphere-practical-verified-kernels-with-rust-and-verus.md)** — 把可验证内核做成可工程化:raw-pointer Rust 配 Verus,加上 flat permission map 与显式分配,证明了真实 separation kernel 的 refinement、安全与不干涉。
- **[Tiga](../papers/sosp-2025/tiga-accelerating-geo-distributed-transactions-with-synchronized-clocks.md)** — 一个干净的地理分布 OLTP 原语:同步时钟加一向时延让大多数冲突事务在 1 WRTT 内提交,失败也有清晰的 fallback。
- **[Dandelion](../papers/sosp-2025/unlocking-true-elasticity-for-the-cloud-native-era-with-dandelion.md)** — 重新思考 serverless 弹性:把编程模型缩成纯计算 + 受信通信函数 DAG,把 per-request cold start 做到几百微秒。
