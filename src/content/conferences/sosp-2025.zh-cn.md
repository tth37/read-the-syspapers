SOSP '25 在首尔共收录 **66 篇**论文,ML 系统这回不再是配角,而成了第一大类——66 篇里有 17 篇直接做 LLM serving 或训练基础设施,另有 6 篇是 GPU 与加速器层面的底座工作,几乎也都是为 ML 而生。经典 SOSP 的骨架仍在:内核、可验证系统、内存与存储、分布式数据库、调度,一个都没少;不过整份 program 的重心明显偏向了 *为 ML workload 虚拟化 GPU 与远端内存的底层 substrate*。另一条更安静的线索贯穿始终——形式验证正从英雄式工程变成日常工具,内核、eBPF 扩展、hypervisor、训练方案、文件系统、WASI 运行时里都能看到它的身影。于是,今年这份 program 里最鲜明的两组张力,一组出在 ML 这边的「substrate vs. serving trick」,另一组出在可靠性这边的「verify-once vs. check-at-runtime」。

## 主题

- **LLM serving 与 GPU substrate 正在收敛到一组共享原语。**[Aegaeon](../papers/sosp-2025/aegaeon-effective-gpu-pooling-for-concurrent-llm-serving-on-the-market.md) 把自动伸缩下沉到 token 边界,让 marketplace 上的模型共享同一块 GPU;[Jenga](../papers/sosp-2025/jenga-effective-memory-management-for-serving-llm-with-heterogeneity.md) 和 [DiffKV](../papers/sosp-2025/diffkv-differentiated-memory-management-for-large-language-models-with-parallel-kv.md) 针对异构模型和 K/V 的不对称敏感性,各自重构了 KV 内存布局;[Pie](../papers/sosp-2025/pie-a-programmable-serving-system-for-emerging-llm-applications.md) 把 KV pages 和生成步骤直接暴露成一种叫 inferlet 的编程模型;[PrefillOnly](../papers/sosp-2025/prefillonly-an-inference-engine-for-prefill-only-workloads-in-large-language-model.md) 则干脆专攻单 token 的 prefill 请求;[LithOS](../papers/sosp-2025/lithos-an-operating-system-for-efficient-machine-learning-on-gpus.md) 更进一步,把整套控制面压到 CUDA driver 边界之下,以 TPC 为粒度做调度并自动 kernel atomization。摆在一起看,它们在讲同一件事:serving 系统要的不再是零散 scheduler,而是一整套 OS 式的 substrate。

- **检索、动态性与异构硬件把 LLM 推出了同构 GPU 的舒适区。**[HedraRAG](../papers/sosp-2025/hedrarag-co-optimizing-generation-and-retrieval-for-heterogeneous-rag-workflows.md) 和 [METIS](../papers/sosp-2025/metis-fast-quality-aware-rag-systems-with-configuration-adaptation.md) 把检索和生成放在一起协同优化;[IC-Cache](../papers/sosp-2025/ic-cache-efficient-large-language-model-serving-via-in-context-caching.md) 复用历史 request-response 对作为 in-context demonstration,让请求在大小模型之间路由;[KTransformers](../papers/sosp-2025/ktransformers-unleashing-the-full-potential-of-cpu-gpu-hybrid-inference-for-moe-models.md) 把大型 MoE 推理铺在 CPU AMX 和 GPU 之间一起跑;[HeteroInfer](../papers/sosp-2025/characterizing-mobile-soc-for-accelerating-heterogeneous-llm-inference.md) 把 LLM 的工作量切到手机 SoC 上的 GPU 和 NPU;[Tempo](../papers/sosp-2025/tempo-compiled-dynamic-deep-learning-with-symbolic-dependence-graphs.md) 干脆把「时间」提升为 tensor 的显式维度,用来承接动态的 LLM/RL 负载。

- **训练基础设施已经硬化成独立的一类系统工作。**[ByteRobust](../papers/sosp-2025/robust-llm-training-infrastructure-at-bytedance.md) 完整地讲述了一个 10k-GPU 级别生产栈如何以「隔离优先」处理故障;[Mycroft](../papers/sosp-2025/mycroft-tracing-dependencies-in-collective-communication-towards-reliable-llm-training.md) 通过追踪集合通信依赖,把 hang 的定位压到秒级;[Sailor](../papers/sosp-2025/sailor-automating-distributed-training-over-dynamic-heterogeneous-and-geo-distributed.md) 在资源频繁抖动的情况下,把 allocation、3D 并行与 zone 放置一并选出来;[DCP](../papers/sosp-2025/dcp-addressing-input-dynamism-in-long-context-training-via-dynamic-context-parallelism.md) 抛弃固定的 context-parallel 布局,改为按 batch 动态放置 block;[TrainVerify](../papers/sosp-2025/trainverify-equivalence-based-verification-for-distributed-llm-training.md) 则直接证明分布式训练方案与逻辑模型等价。说到底,可靠性、验证和调度在这个场景里是一回事。

- **形式验证跨出了工具箱,进了内核、verifier 和 hypervisor。**[Atmosphere](../papers/sosp-2025/atmosphere-practical-verified-kernels-with-rust-and-verus.md) 用 Rust+Verus,配上 raw-pointer 代码和 flat permission map,证明了一个功能完整的微内核;[BCF](../papers/sosp-2025/prove-it-to-the-kernel-precise-extension-analysis-via-proof-guided-abstraction-refinement.md) 把 eBPF 里那些最难的 refinement 证明甩到用户态,再在内核里用线性时间核对,一举救回了 403 个被误判拒绝的程序;[TickTock](../papers/sosp-2025/ticktock-verified-isolation-in-a-production-embedded-os.md) 用 Flux 把 Tock 原本笼统的 MPU 抽象换成细粒度的、可证明的版本;[CortenMM](../papers/sosp-2025/cortenmm-efficient-memory-management-with-strong-correctness-guarantees.md) 给 range-transactional 内存管理器的锁定核心加上证明;[Ghost in the Android Shell](../papers/sosp-2025/ghost-in-the-android-shell-pragmatic-test-oracle-specification-of-a-production-hypervisor.md) 把可执行的 ghost-state 规范直接嵌进了 pKVM;[AutoMan](../papers/sosp-2025/automan-facilitating-verified-distributed-systems-development-through-automatic-code.md) 则从 Dafny action 出发自动生成可验证代码,把热路径留给开发者自己调。

- **内存继续朝多层、远端池和相互依赖的资源方向解构。**[Scalable Far Memory](../papers/sosp-2025/scalable-far-memory-balancing-faults-and-evictions.md) 彻底把 eviction 从 fault 路径上解耦;[Spirit](../papers/sosp-2025/spirit-fair-allocation-of-interdependent-resources-in-remote-memory-systems.md) 把本地缓存和远端带宽放在同一个定价框架里;[Demeter](../papers/sosp-2025/demeter-a-scalable-and-elastic-tiered-memory-solution-for-virtualized-cloud-via-guest.md) 把 VM 的分层逻辑下放进 guest,由 guest 自己驱动 PEBS;[cache_ext](../papers/sosp-2025/cache-ext-customizing-the-page-cache-with-ebpf.md) 把 per-cgroup 的替换策略藏到一个 eBPF 接口后面;[How to Copy Memory](../papers/sosp-2025/how-to-copy-memory-coordinated-asynchronous-copy-as-a-first-class-os-service.md) 把内存拷贝改造成由 OS 统一编排的异步服务;[Scalable Address Spaces](../papers/sosp-2025/scalable-address-spaces-using-concurrent-interval-skiplist.md) 则用并发区间 skiplist 取代了 mmap 那把粗糙的大锁。

## 显著趋势

- **把策略点下沉到 vendor driver 或 framework 的下方。**LLM 和 ML 系统反复在做同一件事:把控制面塞进一个已经固化的 API 之下。[LithOS](../papers/sosp-2025/lithos-an-operating-system-for-efficient-machine-learning-on-gpus.md) 在 CUDA driver API 上做 interpose;[PhoenixOS](../papers/sosp-2025/phoenixos-concurrent-os-level-gpu-checkpoint-and-restore-with-validated-speculation.md) 用「先推测、后核对」的访问集,做 OS 级并发的 GPU C/R;[Mercury](../papers/sosp-2025/mercury-unlocking-multi-gpu-operator-optimization-for-llms-via-remote-memory-scheduling.md) 把 GPU 显存当作可调度的远端存储来编排;[GoFS](../papers/sosp-2025/managing-scalable-direct-storage-accesses-for-gpus-with-gofs.md) 把 F2FS 的元数据和 NVMe 队列控制搬上 GPU;[cache_ext](../papers/sosp-2025/cache-ext-customizing-the-page-cache-with-ebpf.md) 和 [FlexGuard](../papers/sosp-2025/flexguard-fast-mutual-exclusion-independent-of-subscription.md) 把 eBPF hook 变成可定制的内核策略点。套路其实都一样:不 fork 框架、不 fork vendor stack,却依然把控制权攥在自己手里。

- **对 common case 做预测,rare case 再去核对与回退。**[Tiga](../papers/sosp-2025/tiga-accelerating-geo-distributed-transactions-with-synchronized-clocks.md) 给跨地域事务预测一个未来时间戳作为定序,再通过 super-quorum 的日志核对兜底;[Pesto](../papers/sosp-2025/pesto-cooking-up-high-performance-bft-queries.md) 对无冲突的 BFT SQL 直接跳过全序;[PhoenixOS](../papers/sosp-2025/phoenixos-concurrent-os-level-gpu-checkpoint-and-restore-with-validated-speculation.md) 从 launch 参数推测 GPU kernel 的访问集;[Optimistic Recovery (Phoenix)](../papers/sosp-2025/optimistic-recovery-for-high-availability-software-via-partial-process-state-preservation.md) 主进程重启的同时复用一部分长寿命状态;[Orthrus](../papers/sosp-2025/orthrus-efficient-and-timely-detection-of-silent-user-data-corruption-in-the-cloud-with.md) 把数据路径上的闭包搬到别的核上重新校验一遍。照例,每一篇都用一点点回滚/复核成本,换回 common case 的显著收益。

- **与其把编程模型做大,不如把它做小。**[Dandelion](../papers/sosp-2025/unlocking-true-elasticity-for-the-cloud-native-era-with-dandelion.md) 用一张纯计算函数的 DAG 替代 POSIX serverless sandbox;[Quilt](../papers/sosp-2025/quilt-resource-aware-merging-of-serverless-workflows.md) 把那些合并起来更划算的 serverless 子图直接融合成单个 LLVM 级进程;[µFork](../papers/sosp-2025/fork-supporting-posix-fork-within-a-single-address-space-os.md) 在单一 CHERI 地址空间里模拟出 POSIX fork;[Pie](../papers/sosp-2025/pie-a-programmable-serving-system-for-emerging-llm-applications.md) 用 inferlet 取代固定的 LLM serving loop。换句话说,API 越小,runtime 越敢深度特化。

- **让难写的东西从已有 artifact 里生出来。**[KNighter](../papers/sosp-2025/knighter-transforming-static-analysis-with-llm-synthesized-checkers.md) 从 bug-fix patch 出发,再借助 LLM 做筛选,合成出 Clang Static Analyzer checker;[Fawkes](../papers/sosp-2025/fawkes-finding-data-durability-bugs-in-dbmss-via-recovered-data-state-verification.md) 以 DBMS trace 为素材,构造出一张 checkpoint-rectified 状态图;[WASIT](../papers/sosp-2025/wasit-deep-and-continuous-differential-testing-of-webassembly-system-interface.md) 把 WASI 规范直接翻成可执行约束和抽象资源状态;[Veritas](../papers/sosp-2025/ebpf-misbehavior-detection-fuzzing-with-a-specification-based-oracle.md) 则把 eBPF fuzzing 与 Dafny 规范做成的 oracle 成对使用。到头来,invariant 与 oracle 都来自已经存在的代码和规范,而不是临时手写的一套新东西。

## 必读推荐

- **[LithOS](../papers/sosp-2025/lithos-an-operating-system-for-efficient-machine-learning-on-gpus.md)** — 把 GPU OS 这件事讲成一套 substrate,而非又一个 point scheduler:TPC 粒度调度、透明的 kernel atomization、right-sizing 与 DVFS,全部在 CUDA driver 边界下统一起来。
- **[Aegaeon](../papers/sosp-2025/aegaeon-effective-gpu-pooling-for-concurrent-llm-serving-on-the-market.md)** — 关于「在模型 marketplace 上做 token 边界自动伸缩」,这是目前最清晰的一个实测数据点:在阿里云 beta 部署中 GPU 用量降低 82%,可承载的 arrival rate 提升 2-2.5×。
- **[Atmosphere](../papers/sosp-2025/atmosphere-practical-verified-kernels-with-rust-and-verus.md)** — 把可验证内核做到可工程化的程度:raw-pointer Rust 配合 Verus,加上 flat permission map 和显式分配,为一个真实的 separation kernel 证明了 refinement、安全性与不干涉。
- **[Tiga](../papers/sosp-2025/tiga-accelerating-geo-distributed-transactions-with-synchronized-clocks.md)** — 一件很干净的跨地域 OLTP 原语:同步时钟再加上实测单向时延,大多数冲突事务都能在 1 WRTT 内提交,偶尔失败也有定义良好的 validation 回退。
- **[Dandelion](../papers/sosp-2025/unlocking-true-elasticity-for-the-cloud-native-era-with-dandelion.md)** — 重新审视 serverless 弹性:把编程模型压缩成「纯计算 + 受信通信」的 DAG,每个请求的 cold start 直接做到几百微秒。

## 数据

- **覆盖论文:** 66 篇(整份 program)。
- **类别分布:** llm-serving 11、storage-and-databases 10、verification-and-reliability 10、datacenter-scheduling 10、memory-systems 7、llm-training-infra 6、gpu-and-accelerator-systems 6、embedded-os-and-security 6。

- **工业界参与(至少一篇):** 阿里、字节、Google、华为、Meta、Microsoft Research、NVIDIA、三星、腾讯,另有几篇关于已上线系统的回顾——[Tock](../papers/sosp-2025/tock-from-research-to-securing-10-million-computers.md)、[ByteRobust](../papers/sosp-2025/robust-llm-training-infrastructure-at-bytedance.md)、[CHERIoT RTOS](../papers/sosp-2025/cheriot-rtos-an-os-for-fine-grained-memory-safe-compartments-on-low-cost-embedded-devices.md),都已经跑到了生产规模。
- **教学与 artifact-evaluation 类论文:** [Proto](../papers/sosp-2025/proto-a-guided-journey-through-modern-os-construction.md) 在 Raspberry Pi 上带出一套五阶段、以应用驱动的 OS 构建课程;[Analyzing and Enhancing ArckFS](../papers/sosp-2025/analyzing-and-enhancing-arckfs-an-anecdotal-example-of-benefits-of-artifact-evaluation.md) 则通过 artifact evaluation 顺手修掉了真实 bug。顺带一提,这两篇也说明 SOSP 仍愿意为系统社区的基础设施留出版面,不是只看新机制。
