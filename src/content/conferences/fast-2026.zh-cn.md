FAST '26 这届在 Santa Clara 收了 **44 篇**,议程几乎被存储一口气吞下,不过真正有意思的是,整届被两股力道同时拉扯着走:一边是 AI 负载正重新勾勒存储路径的走向,另一边是新一代设备——CXL-SSD、FDP、zoned UFS、高带宽全闪阵列——已经跑到了本该服务它们的 Linux 时代 I/O 栈前面。今年来自 Apple、Alibaba、ByteDance、Samsung、Huawei、SK hynix 等厂商的工业界系统论文占比高得不寻常。FAST 历来对产业友好,但这一届尤其偏向生产规模下的成本、可靠性与部署工程。

## 主题

**AI 正在重画整条存储栈。** 44 篇里有 8 篇明确落在 AI-era-storage 这一条线上,还有若干从侧面擦边。共同的判断是:LLM 和 ML 负载不只是新来的租户,它们会反过来逼着存储路径重新设计。[SolidAttention](../papers/fast-2026/solidattention-low-latency-ssd-based-serving-on-memory-constrained-pcs.md) 把稀疏注意力、SSD 布局与 DAG 调度拧在一起,专门服务本地 128k 推理;[CacheSlide](../papers/fast-2026/cacheslide-unlocking-cross-position-aware-kv-cache-reuse-for-accelerating-llm-serving.md) 和 [Bidaw](../papers/fast-2026/bidaw-enhancing-key-value-caching-for-interactive-llm-serving-via-bidirectional-computatio.md) 则瞄着云端推理,分别追跨位置 KV 复用与双向 compute-storage 感知的 KV 放置;[面向 LLM 模型加载的可编程 page cache](../papers/fast-2026/accelerating-model-loading-in-llm-inference-by-programmable-page-cache.md) 与 [AITurbo 的 grouped I/O API](../papers/fast-2026/fast-cloud-storage-for-ai-jobs-via-grouped-i-o-api-with-transparent-read-write-optimizatio.md) 则把 AI 形态的访问模式一路反推回存储栈;[AdaCheck](../papers/fast-2026/adacheck-an-adaptive-checkpointing-system-for-efficient-llm-training-with-redundancy-utili.md) 与 [GCR](../papers/fast-2026/gpu-checkpoint-restore-made-fast-and-lightweight.md) 让每一步级别的 LLM 训练 checkpoint 和 GPU checkpoint 都能负担得起;[Seneca](../papers/fast-2026/preparation-meets-opportunity-enhancing-data-preprocessing-for-ml-training-with-seneca.md) 则干脆把 ML 预处理缓存本身重写了。

**为更快、更致密的设备专门定制 I/O 路径。** 第二组论文在重建 Linux 时代的那套原语,好让它们跟上底下硬件的节奏。[Rearchitecting buffered I/O](../papers/fast-2026/rearchitecting-buffered-i-o-in-the-era-of-high-bandwidth-ssds.md) 把贯穿到底的 page cache 换成一块 scrap buffer,专伺候高带宽 SSD 阵列;[UnICom](../papers/fast-2026/unicom-a-universally-high-performant-i-o-completion-mechanism-for-modern-computer-systems.md) 把 direct-I/O 的完成搬进内核;[DPAS](../papers/fast-2026/dpas-a-prompt-accurate-and-safe-i-o-completion-method-for-ssds.md) 给每个 I/O 学出合适的睡眠时长;[ScaleSwap](../papers/fast-2026/scaleswap-a-scalable-os-swap-system-for-all-flash-swap-arrays.md) 围绕每核文件和委托式元数据重做 swap;[uCache](../papers/fast-2026/ucache-a-customizable-unikernel-based-io-cache.md) 让 `mmap` 风格的 IO 缓存在 unikernel 里变得可定制;[ZUFS](../papers/fast-2026/unleashing-zoned-ufs-cross-layer-optimizations-for-next-generation-mobile-storage.md) 终于把 zoned UFS 做到手机上能发货;而 [Advancing Data Integrity in Linux](../papers/fast-2026/advancing-data-integrity-in-linux.md) 则把 PI 一路推到 io_uring 和文件系统层。

**在硅片出厂前就仿真它。** CXL 和下一代 SSD 规范大多还停在纸面,本届 FAST 难得让这样一条子线独立成型:让研究者在芯片真正到手之前就能研究这些设备。[Xerxes](../papers/fast-2026/xerxes-extensive-exploration-of-scalable-hardware-systems-with-cxl-based-simulation-framew.md) 仿真带 device-managed coherence 的 CXL 3.1 fabric;[Cylon](../papers/fast-2026/cylon-fast-and-accurate-full-system-emulation-of-cxl-ssds.md) 重映射 EPT 表项,让 CXL-SSD 命中走成直接 load;[WARP](../papers/fast-2026/characterizing-and-emulating-fdp-ssds-with-warp.md) 对 FDP SSD 做刻画,把控制器里那些藏起来的决策翻出来,暴露成显式的 knob。

**规模化的工业成本工程。** FAST 一向有产业味,今年尤其密集。[McQueen](../papers/fast-2026/mcqueen-apple-s-geo-distributed-object-store-at-exabyte-scale.md) 借五区 XOR segmentation 把 Apple 对象存储的副本系数从 2.40 压到 1.50;[PolarStore](../papers/fast-2026/polarstore-high-performance-data-compression-for-large-scale-cloud-native-databases.md) 把 16 KB 软件压缩和字节粒度 CSD 压缩叠起来,让 PolarDB 的存储成本下降 60%;[DisCoGC](../papers/fast-2026/discard-based-garbage-collection-for-distributed-log-structured-storage-systems-in-bytedan.md) 用轻量 discard 在日志结构的 ByteStore 上把 ByteDance 的生产 TCO 砍掉大约 20%;[TapeOBS](../papers/fast-2026/cost-efficient-archive-cloud-storage-with-tape-design-and-deployment.md) 建模了一条先过 HDD 再落磁带的归档方案,十年 TCO 最多降 4.95x;[RASK](../papers/fast-2026/range-as-a-key-is-the-key-fast-and-compact-cloud-block-store-index-with-rask.md) 把云块存储索引的内存砍掉 98.9%;同一条线上还有一篇偏综述性质的[关于「云中的本地存储」过去、现在与未来的回顾](../papers/fast-2026/here-there-and-everywhere-the-past-the-present-and-the-future-of-local-storage-in-cloud.md),为下一代提了个 local-plus-EBS 的混合方案。

**日志结构与纠删码系统里更聪明的放置和修复。** 还有一组论文不动形态,只换策略。[DOGI](../papers/fast-2026/dogi-data-placement-with-oracle-guided-insights-for-log-structured-systems.md) 给日志结构系统引入 oracle 风格的热数据过滤器;[HATS](../papers/fast-2026/holistic-and-automated-task-scheduling-for-distributed-lsm-tree-based-storage.md) 把副本分配、请求重路由和读驱动的 compaction 合成一个统一调度;[DRBoost](../papers/fast-2026/drboost-boosting-degraded-read-performance-in-msr-coded-storage-clusters.md) 在 MSR 编码存储上只重建真正需要的那些 sub-chunk;[LESS](../papers/fast-2026/less-is-more-for-i-o-efficient-repairs-in-erasure-coded-storage.md) 叠几条 RS 编码的扩展 sub-stripe,把 EC 修复 I/O 压下去,又不必像 Clay 那样付出 seek 暴涨的代价;至于 [ParaSync](../papers/fast-2026/parasync-exploiting-fine-grained-parallelism-for-efficient-file-synchronization.md) 和 [SkySync](../papers/fast-2026/skysync-accelerating-file-synchronization-with-collaborative-delta-generation.md) 则为 CDC 风格的文件同步做并行化和 checksum 复用。

## 值得关注的趋势

- **块级稀疏注意力正在被搬到 SSD 上。** 至少三篇论文——[SolidAttention](../papers/fast-2026/solidattention-low-latency-ssd-based-serving-on-memory-constrained-pcs.md)、[CacheSlide](../papers/fast-2026/cacheslide-unlocking-cross-position-aware-kv-cache-reuse-for-accelerating-llm-serving.md) 和 [Bidaw](../papers/fast-2026/bidaw-enhancing-key-value-caching-for-interactive-llm-serving-via-bidirectional-computatio.md)——都在说同一件事:注意力的选择粒度和存储的传输粒度本就不该是同一个东西;只有把这两者共同设计,128k 上下文的 decode 才可能在 KV 不常驻内存的情况下跑起来。
- **头条被生产部署抢走了。** Apple、Alibaba、ByteDance 和 Samsung 各自贡献了规模化的系统论文:[McQueen](../papers/fast-2026/mcqueen-apple-s-geo-distributed-object-store-at-exabyte-scale.md)、[PolarStore](../papers/fast-2026/polarstore-high-performance-data-compression-for-large-scale-cloud-native-databases.md)、[DisCoGC](../papers/fast-2026/discard-based-garbage-collection-for-distributed-log-structured-storage-systems-in-bytedan.md),以及做阈值挖掘的 [FailureMiner](../papers/fast-2026/failureminer-a-joint-key-decision-mining-scheme-for-practical-ssd-failure-prediction-and-a.md)。今年工业主导论文的比例明显高于一般纯学术系统会议。
- **CXL 与 FDP 这一类设备,先仿真,再谈评估。** [Xerxes](../papers/fast-2026/xerxes-extensive-exploration-of-scalable-hardware-systems-with-cxl-based-simulation-framew.md)、[Cylon](../papers/fast-2026/cylon-fast-and-accurate-full-system-emulation-of-cxl-ssds.md) 和 [WARP](../papers/fast-2026/characterizing-and-emulating-fdp-ssds-with-warp.md) 都把仿真本身当作一等的研究产出,折射出这些规范跑在出货硅片前面有多远。
- **每核、每请求的原语,把全局调度器顶了下去。** [ScaleSwap](../papers/fast-2026/scaleswap-a-scalable-os-swap-system-for-all-flash-swap-arrays.md) 把 swap 按核分片,[UnICom](../papers/fast-2026/unicom-a-universally-high-performant-i-o-completion-mechanism-for-modern-computer-systems.md) 用调度器打 tag 的完成路径,[Lockify](../papers/fast-2026/lockify-understanding-linux-distributed-lock-management-overheads-in-shared-storage.md) 干脆让文件创建者自行声明初始锁的 owner,而不必再去碰 DLM。说到底,设备一旦够快,瓶颈就总会回到 OS 里那条共享路径上。

## 必读推荐

- **[SolidAttention: Low-Latency SSD-based Serving on Memory-Constrained PCs](../papers/fast-2026/solidattention-low-latency-ssd-based-serving-on-memory-constrained-pcs.md)** — 整届最干净的一例「AI × 存储」共同设计:interleaved KV 布局、带 overwrite correction 的投机预取,再加上 DAG 调度,本地 128k 推理上量化出 3.1x 吞吐和约 98% 的 KV 内存削减。
- **[McQueen: Apple's Geo-Distributed Object Store at Exabyte Scale](../papers/fast-2026/mcqueen-apple-s-geo-distributed-object-store-at-exabyte-scale.md)** — 真·exabyte 规模的生产部署,从双区复制演进到五区 XOR segmentation,副本系数从 2.40 降到 1.50。这种端到端的工业设计论文本身就不多见。
- **[PolarStore: High-Performance Data Compression for Large-Scale Cloud-Native Databases](../papers/fast-2026/polarstore-high-performance-data-compression-for-large-scale-cloud-native-databases.md)** — Alibaba Cloud 对「PolarDB 的存储成本到底花在哪里」给出的答案:16 KB 软件压缩叠字节粒度 CSD 压缩,再配上数据库特化的快路径,拿到 3.55x 压缩比和大约 60% 的存储成本下降。
- **[Advancing Data Integrity in Linux](../papers/fast-2026/advancing-data-integrity-in-linux.md)** — Samsung 与 EPFL 在上游 Linux 上的工作,把 PI 延伸到 io_uring 元数据路径,以及文件系统管理的 PI;其影响面很可能远超论文本身给出的那几个 benchmark。
- **[Xerxes: Extensive Exploration of Scalable Hardware Systems with CXL-Based Simulation Framework](../papers/fast-2026/xerxes-extensive-exploration-of-scalable-hardware-systems-with-cxl-based-simulation-framew.md)** — 面向 CXL 3.1 fabric、带 device-managed coherence 的社区基础设施;未来一两年里,应该会有不少存储与内存方向的论文直接搭在它上面。
