本届 FAST '26 在 Santa Clara 共录用 **44 篇论文**——议程压倒性地偏向存储，
但这种存储叙事被当下两股压力深刻塑造：AI 负载正在重塑存储路径；而新型
设备（CXL-SSD、FDP、zoned UFS、高带宽全闪阵列）跑得比那套 Linux 时代的
I/O 栈更快。Apple、Alibaba、ByteDance、Samsung、Huawei、SK hynix 等产业
界的规模化系统论文占据了不寻常的比例——FAST 一直对产业友好，但本届尤其
偏重生产级别的成本、可靠性与部署工程。

## 主题

**AI 正在重画存储栈。** 44 篇论文中有 8 篇直接属于 AI-era-storage 赛道，另有数篇从侧面切入。共同的判断是：LLM / ML 负载并不只是新租户，而是倒逼整个存储路径重设计。[SolidAttention](../papers/fast-2026/solidattention-low-latency-ssd-based-serving-on-memory-constrained-pcs.md) 把稀疏注意力、SSD 友好布局和 DAG 调度一起共设，目标是本地 128k 推理；[CacheSlide](../papers/fast-2026/cacheslide-unlocking-cross-position-aware-kv-cache-reuse-for-accelerating-llm-serving.md) 和 [Bidaw](../papers/fast-2026/bidaw-enhancing-key-value-caching-for-interactive-llm-serving-via-bidirectional-computatio.md) 则面向云侧推理，分别追求跨位置 KV 复用和双向 compute-storage 感知的 KV 放置；[programmable page cache 用于 LLM 模型加载](../papers/fast-2026/accelerating-model-loading-in-llm-inference-by-programmable-page-cache.md)、[AITurbo 的 grouped I/O API](../papers/fast-2026/fast-cloud-storage-for-ai-jobs-via-grouped-i-o-api-with-transparent-read-write-optimizatio.md) 把 AI 形态的访问模式推回存储栈；[AdaCheck](../papers/fast-2026/adacheck-an-adaptive-checkpointing-system-for-efficient-llm-training-with-redundancy-utili.md) 和 [GCR](../papers/fast-2026/gpu-checkpoint-restore-made-fast-and-lightweight.md) 让 per-step LLM 训练与 GPU checkpoint 变得可承担；[Seneca](../papers/fast-2026/preparation-meets-opportunity-enhancing-data-preprocessing-for-ml-training-with-seneca.md) 则直接重写 ML 预处理缓存。

**为更快、更致密的设备专门优化 I/O 路径。** 第二类论文在重建 Linux 时代的原语，让它们追上底层设备的速度。[Rearchitecting buffered I/O](../papers/fast-2026/rearchitecting-buffered-i-o-in-the-era-of-high-bandwidth-ssds.md) 用 scrap buffer 替代"一路 page cache"的缓冲写路径，面向高带宽 SSD 阵列；[UnICom](../papers/fast-2026/unicom-a-universally-high-performant-i-o-completion-mechanism-for-modern-computer-systems.md) 把 direct-I/O 完成逻辑搬进内核；[DPAS](../papers/fast-2026/dpas-a-prompt-accurate-and-safe-i-o-completion-method-for-ssds.md) 为 SSD 学习 per-I/O 睡眠时长；[ScaleSwap](../papers/fast-2026/scaleswap-a-scalable-os-swap-system-for-all-flash-swap-arrays.md) 用 per-core swap 文件与委托式元数据重构 swap；[uCache](../papers/fast-2026/ucache-a-customizable-unikernel-based-io-cache.md) 在 unikernel 里做出可定制的 `mmap` 式 IO 缓存；[ZUFS](../papers/fast-2026/unleashing-zoned-ufs-cross-layer-optimizations-for-next-generation-mobile-storage.md) 让 zoned UFS 第一次真正可以上手机；[Advancing Data Integrity in Linux](../papers/fast-2026/advancing-data-integrity-in-linux.md) 把 PI 一路推到 io_uring 和文件系统层。

**仿真尚未出货的硬件。** CXL 和下一代 SSD 规范很多还停留在纸面，本届 FAST 不寻常地有一整条主题在为研究者提供"在硅片到来之前就能研究这些设备"的工具链。[Xerxes](../papers/fast-2026/xerxes-extensive-exploration-of-scalable-hardware-systems-with-cxl-based-simulation-framew.md) 仿真带 device-managed coherence 的 CXL 3.1 fabric；[Cylon](../papers/fast-2026/cylon-fast-and-accurate-full-system-emulation-of-cxl-ssds.md) 通过重映射 EPT，让 CXL-SSD 命中走成直接 load；[WARP](../papers/fast-2026/characterizing-and-emulating-fdp-ssds-with-warp.md) 系统刻画 FDP SSD，并把控制器里那些看不见的选择变成显式的 knob。

**规模化的工业成本工程。** FAST 一直有产业味，但今年尤其密集。[McQueen](../papers/fast-2026/mcqueen-apple-s-geo-distributed-object-store-at-exabyte-scale.md) 把 Apple 对象存储从双区复制迁到五区 XOR segmentation，副本系数从 `2.40` 降到 `1.50`；[PolarStore](../papers/fast-2026/polarstore-high-performance-data-compression-for-large-scale-cloud-native-databases.md) 用 `16 KB` 软件压缩加上 byte 粒度 CSD 压缩，给 PolarDB 带来 `3.55` 压缩比和约 `60%` 的存储成本下降；[DisCoGC](../papers/fast-2026/discard-based-garbage-collection-for-distributed-log-structured-storage-systems-in-bytedan.md) 在 ByteDance 生产 ByteStore 上用轻量 discard 把 TCO 降低约 `20%`；[TapeOBS](../papers/fast-2026/cost-efficient-archive-cloud-storage-with-tape-design-and-deployment.md) 以 HDD 作为 staging 把磁带做成云归档后端，十年 TCO 最高降 `4.95x`；[RASK](../papers/fast-2026/range-as-a-key-is-the-key-fast-and-compact-cloud-block-store-index-with-rask.md) 把云块存储索引内存砍掉 `98.9%`；同一赛道里还有一篇偏综述性的[关于"云中的本地存储"过去/现在/未来的回顾](../papers/fast-2026/here-there-and-everywhere-the-past-the-present-and-the-future-of-local-storage-in-cloud.md)，提出 local-plus-EBS 的下一代混合方案。

**日志结构与纠删码系统里的更聪明的放置与修复。** 还有一批论文是在已经成熟的存储形态上做更好的策略。[DOGI](../papers/fast-2026/dogi-data-placement-with-oracle-guided-insights-for-log-structured-systems.md) 为日志结构系统做 oracle 风格的 hot filter 放置；[HATS](../papers/fast-2026/holistic-and-automated-task-scheduling-for-distributed-lsm-tree-based-storage.md) 把副本分配、请求再路由、读驱动 compaction 合成一个调度器；[DRBoost](../papers/fast-2026/drboost-boosting-degraded-read-performance-in-msr-coded-storage-clusters.md) 在 MSR 编码存储里只重建真正需要的 sub-chunk；[LESS](../papers/fast-2026/less-is-more-for-i-o-efficient-repairs-in-erasure-coded-storage.md) 用几条 RS 扩展 sub-stripe 把 EC 修复 I/O 压下去，而不必像 Clay 那样付出 seek 爆炸；[ParaSync](../papers/fast-2026/parasync-exploiting-fine-grained-parallelism-for-efficient-file-synchronization.md) 和 [SkySync](../papers/fast-2026/skysync-accelerating-file-synchronization-with-collaborative-delta-generation.md) 则为 CDC 风格文件同步做并行化和 checksum 复用。

## 值得关注的趋势

- **块级稀疏注意力正在"搬到 SSD 上"。** 至少三篇论文——[SolidAttention](../papers/fast-2026/solidattention-low-latency-ssd-based-serving-on-memory-constrained-pcs.md)、[CacheSlide](../papers/fast-2026/cacheslide-unlocking-cross-position-aware-kv-cache-reuse-for-accelerating-llm-serving.md) 和 [Bidaw](../papers/fast-2026/bidaw-enhancing-key-value-caching-for-interactive-llm-serving-via-bidirectional-computatio.md)——主张：注意力的选择粒度和存储的传输粒度不应是同一个东西，共设这两个维度之后，`128k` 级上下文 decode 才能在 KV 不完全常驻内存的前提下工作。
- **生产部署级的论文正在 headlining。** Apple、Alibaba、ByteDance、Samsung 各自贡献了规模化系统论文——[McQueen](../papers/fast-2026/mcqueen-apple-s-geo-distributed-object-store-at-exabyte-scale.md)、[PolarStore](../papers/fast-2026/polarstore-high-performance-data-compression-for-large-scale-cloud-native-databases.md)、[DisCoGC](../papers/fast-2026/discard-based-garbage-collection-for-distributed-log-structured-storage-systems-in-bytedan.md)、[FailureMiner](../papers/fast-2026/failureminer-a-joint-key-decision-mining-scheme-for-practical-ssd-failure-prediction-and-a.md)——工业主导论文的比例明显高于纯学术系统会议的水平。
- **CXL 和 FDP 类设备先做仿真，再做评估。** [Xerxes](../papers/fast-2026/xerxes-extensive-exploration-of-scalable-hardware-systems-with-cxl-based-simulation-framew.md)、[Cylon](../papers/fast-2026/cylon-fast-and-accurate-full-system-emulation-of-cxl-ssds.md) 和 [WARP](../papers/fast-2026/characterizing-and-emulating-fdp-ssds-with-warp.md) 都把仿真/刻画作为一等研究产出，这反映出这些规范跑在出货硅片前面的幅度。
- **用 per-core / per-request 原语替换全局调度器。** [ScaleSwap](../papers/fast-2026/scaleswap-a-scalable-os-swap-system-for-all-flash-swap-arrays.md) 把 swap 按核切分；[UnICom](../papers/fast-2026/unicom-a-universally-high-performant-i-o-completion-mechanism-for-modern-computer-systems.md) 用 scheduler tag 做完成路径；[Lockify](../papers/fast-2026/lockify-understanding-linux-distributed-lock-management-overheads-in-shared-storage.md) 让文件创建者直接声明初始锁 owner。当设备足够快之后，瓶颈几乎总在 OS 的共享路径上。

## 必读推荐

- **[SolidAttention: Low-Latency SSD-based Serving on Memory-Constrained PCs](../papers/fast-2026/solidattention-low-latency-ssd-based-serving-on-memory-constrained-pcs.md)** — 全场最干净的 AI × 存储共设：interleaved KV 布局、speculative prefetch + overwrite correction、DAG 调度；本地 `128k` 推理定量给出 `3.1x` 吞吐和约 `98%` 的 KV 内存下降。
- **[McQueen: Apple's Geo-Distributed Object Store at Exabyte Scale](../papers/fast-2026/mcqueen-apple-s-geo-distributed-object-store-at-exabyte-scale.md)** — 真正 exabyte 规模的生产系统，从双区复制演化到五区 XOR segmentation，副本系数 `2.40 → 1.50`；端到端的工业系统设计论文少见。
- **[PolarStore: High-Performance Data Compression for Large-Scale Cloud-Native Databases](../papers/fast-2026/polarstore-high-performance-data-compression-for-large-scale-cloud-native-databases.md)** — Alibaba Cloud 针对"PolarDB 的存储成本到底去哪儿了"的答卷：`16 KB` 软件压缩 + 字节粒度 CSD 压缩 + 数据库特化快路径，`3.55x` 压缩比，约 `60%` 的存储成本下降。
- **[Advancing Data Integrity in Linux](../papers/fast-2026/advancing-data-integrity-in-linux.md)** — Samsung / EPFL 的上游 Linux 工作，把 PI 扩展到 io_uring 元数据路径和文件系统管理的 PI，其影响范围很可能超过论文本身的 benchmark。
- **[Xerxes: Extensive Exploration of Scalable Hardware Systems with CXL-Based Simulation Framework](../papers/fast-2026/xerxes-extensive-exploration-of-scalable-hardware-systems-with-cxl-based-simulation-framew.md)** — 面向 CXL 3.1 fabric 的社区基础设施（含 device-managed coherence），未来一两年里很多存储 / 内存论文都会搭在它上面。
